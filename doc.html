Documentation

What I did

My Recipe Website has a simple purpose. It serves to let users input an ingredient or multiple ingredients and look up possible dishes that they can make with what they have. For instance, if I have "bubble" (although I'm not sure what kind of ingredient this is) and "tea", I might be interested in knowing whether I can make bubble tea.

Why I did it

My girlfriend is studying abroad right now...and she has to cook for herself. So when I asked her for some ideas, she suggested this. I was looking for something that would be useful, so I really took to the idea.

How I did it

I started out with some ideas in mind. The webiste would be a one-page landing site. I would have the search bar very conspicuously positioned, so that the user knows clearly that he or she must start by typing something in. I've succesfully created some SVG animation for the search bar that automatically runs when the pages loads, but I haven't put it on this current version yet because some details still need to be tweaked when the search happens.

After having a rough idea of what the website would look like, I set about looking for a background. That wasn't too hard.

The most challenging part was finding a suitable API. I needed one that allowed recipe search by ingredient(s). Ideally, the API will give me the recipe information, and not just the link to the recipe websites. It took a long time to find Spoonacular on Mashapes.com. Although it is limited to 50 calls/day for free users, it provided extensive data, including what I needed.

Resources leveraged

Google.I Googled almost everything. From how to position something dead center on the page to how I can animate my search bar to become a fixed nav bar on the top of the page. Here's a link to Google.com in case you've never used it before.

Some things I found were...

  • How to make the search bar expand.

  • Creating a translucent box on the page.

  • I used a font recommended by our good classmate Rick.

I used websites like w3schools and jQuery.com extensively.

Major challenges/realizations

My main challenge came from the asynchronous nature of AJAX. I had to ensure that certain chunks of my JavaScript executed only after certain AJAX calls happened. Also, I had to bind certain calls in order to create an array that has the information in the right order. Dealing with these were tough, and the only solution I found was to pollute the global scope a bit and create some variables outside of functions so that I can keep count on when and how many times a call is happening.

A challenge I am still struggling with is how I can display the recipes as image and titles after a search is entered. I am trying to figure out a good grid layout for the images so that they can be clearly seen. Ideally, I also want to display the recipe information on my own website instead of linking to an external one. But that is work for some other time.

Lessons learned

Better planning makes the actual working process much easier. Think through what you are going to do first. And then test everything out and make mistakes and fix them and continue to build on your website.
